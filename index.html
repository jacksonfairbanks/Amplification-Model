Key highlights from the run

    Full schedule (monthly)

    At T0 we assume capital is already raised (NAV and notional start at the target ratio). Additional Bitcoin is added to NAV only from T1 onward, when rebalance issuance is used to buy BTC. Chart is drawn from this schedule. Dividends paid monthly on notional; rebalancing uses the formula so new issuance adds to NAV and restores the target amp %.

    Return formulas
    RMSTR = RBTC + (Amp. % / (1 − Amp. %)) × (RBTC − RUSD)
    RMSTR = Expected return of digital equity
    RBTC = Expected BTC ARR (annualized rate of return)
    Amp. % = Digital credit divided by BTC reserve (notional / NAV)
    RUSD = Digital credit dividend rate
